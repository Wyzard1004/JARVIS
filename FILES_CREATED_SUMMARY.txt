================================================================================
API ENDPOINTS SUMMARY
================================================================================

REQUEST COMMANDS:
  POST /api/soldier/{soldier_id}/request-recon
       - Parameters: area_label, target_x, target_y, priority
       - Route: soldier→operator→recon

  POST /api/soldier/{soldier_id}/request-attack
       - Parameters: area_label, target_x, target_y, requires_approval, priority
       - Route: soldier→operator→attack OR soldier→attack

APPROVAL & RELAY:
  POST /api/soldier/{soldier_id}/approve-command/{command_id}
       - Operator approves command and relays to drones
       - Returns: mission_id for tracking

REPORT PROCESSING:
  POST /api/soldier/{soldier_id}/process-recon-report
       - Parameters: mission_id, enemies_detected, coverage_percent, threat_level
       - Route: recon→operator→soldier

  POST /api/soldier/{soldier_id}/process-bda
       - Parameters: mission_id, damage_assessment
       - Processes: targets destroyed/damaged/escaped

STRIKE AUTHORIZATION:
  POST /api/soldier/{soldier_id}/authorize-strike/{recon_report_id}
       - Parameters: priority
       - Route: recon→operator→attack

STATUS & SIMULATION:
  GET /api/soldier/{soldier_id}/status
      - Returns: complete soldier command history and mission tracking

  POST /api/soldier/{soldier_id}/simulate-scenario
       - Parameters: area
       - Returns: full 5-stage tactical scenario
